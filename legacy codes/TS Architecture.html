Min-Cash-Flow Algorithm

The core value proposition: minimizing the number of transactions required to settle debts.

Efficiency Comparison

A naive approach creates N(N-1) transactions. The optimized greedy match algorithm reduces this to N-1 in the best case.

Comparison of transaction count vs group size.

1. Calculate Net Balance

System calculates (Total Paid) - (Total Consumed) for every user.

2. Sort & Separate

Users are divided into Debtors (Negative) and Creditors (Positive) and sorted by magnitude.

3. Greedy Match

The algorithm iteratively matches the largest debtor with the largest creditor to zero out balances efficiently.

Segregated Settlements

Handling "Daily" vs "Major" expenses independently allows for flexible settlement strategies.

Expense Categories

The architecture tags every expense with a `type`. This allows the settlement engine to run on filtered subsets of the data.

  • Major Expenses: Flights, Hotels, Advance bookings. Usually settled post-trip.
  • Daily Expenses: Food, Taxi, Tickets. Can be settled daily to keep cash flowing.
